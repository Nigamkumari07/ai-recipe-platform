## ✨ Features

### 🤖 AI-Powered Features
- **Recipe Generation** — Input ingredients → Gemini AI generates personalized recipes with instructions
- **Pantry Scanning** — Upload fridge/pantry photo → AI detects ingredients using Gemini Vision API
- **Smart Recipe Matching** — Matches ingredients with recipes from TheMealDB and AI-generated options

### 👥 User Management
- **Clerk Authentication** — Secure sign-up/login with OAuth support
- **Subscription Tiers** — Free Plan vs Pro Plan with feature limits
- **User Dashboard** — Track saved recipes, pantry items, preferences

### 🍽️ Content Discovery
- **Cuisine Explorer** — Browse 30+ cuisines (Thai, Italian, Indian, etc.)
- **Category Browsing** — Filter by meal type (Breakfast, Lunch, Dessert, etc.)
- **Recipe Database** — 1000+ recipes from TheMealDB API + AI-generated recipes
- **Save & Bookmark** — Save favorite recipes for quick access

### 🔒 Security & Rate Limiting
- **Arcjet Protection** — API rate limiting & bot protection
- **Tier-Based Limits** — Free users: 5 recipe generations/day, Pro: Unlimited
- **API Token Authentication** — Secure Strapi API access

### 📱 User Experience
- **Responsive Design** — Mobile, tablet, and desktop optimized (Tailwind CSS)
- **Dark Mode Support** — Theme switching with next-themes
- **Rich UI Components** — shadcn/ui + Radix UI primitives
- **Real-time Notifications** — Sonner toast notifications
- **PDF Export** — Download recipes as PDF

## 🚀 Installation & Setup

### Step 1: Clone & Install

```bash
# Clone repository
git clone https://github.com/YOUR_USERNAME/ai-recipe-platform.git
cd ai-recipe-platform-master

# Backend setup
cd backend
npm install

# Frontend setup
cd ../frontend
npm install
```

### Step 2: Configure Backend

Create `backend/.env`:

```env
HOST=0.0.0.0
PORT=1337

# Secrets (generate with: node -e "console.log(require('crypto').randomBytes(16).toString('base64'))")
APP_KEYS="<random-key-1>,<random-key-2>"
API_TOKEN_SALT=<random-key>
ADMIN_JWT_SECRET=<random-key>
TRANSFER_TOKEN_SALT=<random-key>
JWT_SECRET=<random-key>
ENCRYPTION_KEY=<random-key>

# Database (SQLite by default, or PostgreSQL)
# Leave commented for SQLite (no setup needed locally)
# DATABASE_CLIENT=postgres
# DATABASE_URL=postgresql://<user>:<password>@<host>/<dbname>
```

### Step 3: Start Backend & Create Admin

```bash
cd backend
npm run develop
```

- Strapi admin will open at `http://localhost:1337/admin`
- Create admin account (email/password)
- Navigate to **Settings → API Tokens → Create new API Token**
- Select **Full access**, **Unlimited** duration
- **Copy the token value** (appears once)

### Step 4: Configure Frontend

Create `frontend/.env.local`:

```env
# Strapi
NEXT_PUBLIC_STRAPI_URL=http://localhost:1337
STRAPI_API_TOKEN=<token-from-step-3>

# Clerk Auth (sign up at clerk.com)
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=<your-publishable-key>
CLERK_SECRET_KEY=<your-secret-key>

# Gemini AI (get at ai.google.dev)
GEMINI_API_KEY=<your-gemini-key>

# Unsplash Images (get at unsplash.com/developers)
UNSPLASH_ACCESS_KEY=<your-unsplash-key>

# Arcjet Rate Limiting (sign up at arcjet.com)
ARCJET_KEY=<your-arcjet-key>
```

### Step 5: Start Frontend

```bash
cd frontend
npm run dev
```

- App opens at `http://localhost:3000`
- Click **Sign Up** → authenticate with Clerk
- Dashboard loads with available features

## 📊 Database Schema

### 1. Recipe Collection
```json
{
  "title": "string (required)",
  "description": "richtext",
  "cuisine": "enum (italian, chinese, mexican, indian, etc.)",
  "category": "enum (breakfast, lunch, dessert, etc.)",
  "ingredients": "string",
  "instructions": "richtext",
  "prepTime": "integer (minutes)",
  "cookTime": "integer (minutes)",
  "servings": "integer",
  "difficulty": "enum (easy, medium, hard)",
  "imageUrl": "string (URL from Unsplash)",
  "mealdbId": "string (TheMealDB ID, if applicable)",
  "isAiGenerated": "boolean",
  "savedCount": "integer"
}
```

### 2. Pantry Item Collection
```json
{
  "name": "string (required)",
  "quantity": "integer",
  "unit": "enum (kg, g, ml, l, cups, tsp, tbsp)",
  "category": "enum (vegetables, fruits, dairy, grains, etc.)",
  "expiryDate": "date",
  "owner": "relation → User"
}
```

### 3. Saved Recipe Collection
```json
{
  "recipe": "relation → Recipe (required)",
  "user": "relation → User (required)",
  "savedAt": "timestamp",
  "rating": "integer (1-5)",
  "notes": "text"
}
```

### 4. Extended User Schema
```json
{
  "email": "string",
  "username": "string",
  "clerkId": "string (required, unique)",
  "subscriptionTier": "enum (free, pro)",
  "pantryItems": "relation → Pantry Item",
  "savedRecipes": "relation → Saved Recipe"
}
```

## 🔌 API Endpoints

### Recipes
```
GET    /api/recipes                 # List all recipes (paginated)
GET    /api/recipes/:id             # Get single recipe
POST   /api/recipes                 # Create recipe (server action)
PUT    /api/recipes/:id             # Update recipe (server action)
DELETE /api/recipes/:id             # Delete recipe (server action)
```

### Pantry Items
```
GET    /api/pantry-items            # List user's pantry
GET    /api/pantry-items/:id        # Get single item
POST   /api/pantry-items            # Add to pantry (server action)
PUT    /api/pantry-items/:id        # Update item (server action)
DELETE /api/pantry-items/:id        # Remove from pantry (server action)
```

### Saved Recipes
```
GET    /api/saved-recipes           # List user's saved recipes
POST   /api/saved-recipes           # Save recipe (server action)
DELETE /api/saved-recipes/:id       # Unsave recipe (server action)
```

### TheMealDB (Third-party)
```
GET https://www.themealdb.com/api/json/v1/1/categories.php
GET https://www.themealdb.com/api/json/v1/1/list.php?a=list          # Cuisines
GET https://www.themealdb.com/api/json/v1/1/filter.php?a={cuisine}   # By cuisine
```

All responses follow Strapi's standard REST format with pagination, sorting, and filtering support.

## 🎯 Key Features Explained

### 1. **Recipe Generation** (Gemini AI)
**File:** `frontend/actions/recipe.actions.js`

```javascript
// User inputs: "I have chicken, tomato, garlic"
// Gemini generates:
// - Recipe title
// - Ingredients list
// - Step-by-step instructions
// - Cooking time, difficulty, servings
// - Saves to Strapi database
```

**Tier Limits:**
- **Free**: 5 generations/day (Arcjet rate limit)
- **Pro**: Unlimited generations

### 2. **Pantry Scanning** (Gemini Vision)
**File:** `frontend/actions/pantry.actions.js`

```javascript
// User uploads fridge photo
// Gemini Vision API detects:
// - Vegetables, fruits, dairy, grains
// - Quantity estimation
// - Categorization
// - Stores in Pantry Items collection
// - Suggests recipes matching available ingredients
```

### 3. **Subscription Tiers**
**File:** `frontend/components/PricingModal.jsx`

| Feature | Free | Pro |
|---------|------|-----|
| Recipe Generation | 5/day | Unlimited |
| Pantry Scanning | Yes | Yes |
| Save Recipes | Yes | Yes |
| AI Recipe Features | Limited | Full |
| Ad-free | No | Yes |
| Priority Support | No | Yes |

**Enforcement:** `frontend/lib/checkUser.js` validates `user.subscriptionTier`

### 4. **Authentication** (Clerk + Strapi)
**File:** `frontend/lib/checkUser.js`

```javascript
// Flow:
// 1. User signs up with Clerk
// 2. Clerk webhook triggers (optional)
// 3. checkUser() fetches from Strapi using clerkId
// 4. Auto-creates user if first time
// 5. Stores clerkId + subscriptionTier in Strapi
```

### 5. **Rate Limiting** (Arcjet)
**File:** `frontend/lib/arcjet.js`

- **Bot protection**: Detects and blocks bot-like requests
- **Tier-based limits**: Different rate limits for free vs pro users
- **Per-minute caps**: Prevents abuse of AI APIs

## 🚀 Deployment

### Frontend (Vercel - Recommended)

```bash
# Install Vercel CLI
npm i -g vercel

# Deploy
cd frontend
vercel
```

**Environment variables in Vercel:**
- Go to Project Settings → Environment Variables
- Add all values from `.env.local`

### Backend (Strapi Cloud or Self-hosted)

#### Option 1: Strapi Cloud
```bash
cd backend
npm run deploy
# Follow Strapi Cloud authentication
```

#### Option 2: Heroku (free tier ending, but documented)
```bash
# Install Heroku CLI and configure database
heroku create your-app-name
heroku addons:create heroku-postgresql:hobby-dev
git push heroku main
```

#### Option 3: Self-hosted (DigitalOcean, AWS, Railway)
```bash
# Deploy with PostgreSQL database
# Set DATABASE_URL env variable
npm run build && npm run start
```

### Database (Neon PostgreSQL)

1. Create account at [neon.tech](https://neon.tech)
2. Create new project (free tier available)
3. Copy connection string
4. Add to backend `.env`:
   ```env
   DATABASE_CLIENT=postgres
   DATABASE_URL=postgresql://...
   ```
5. Strapi auto-migrates schema on first boot

## 🐛 Troubleshooting

### **"Unable to acquire lock at .next/dev/lock"**
```bash
# Another Next.js instance running
taskkill /F /IM node.exe    # Windows
# or
pkill node                   # Mac/Linux

npm run dev
```

### **"Hydration Mismatch" Error**
- Fixed in `frontend/components/Header.jsx` (removed `SignedIn`/`SignedOut`)
- Use server-side `user` variable for conditional rendering

### **"Duplicate cuisine keys" Warning**
- Fixed in `frontend/app/(main)/dashboard/page.jsx`
- Areas deduplicated before rendering

### **Gemini API Returns Error**
```
❌ Check 1: Is GEMINI_API_KEY set in .env.local?
❌ Check 2: Does key start with "AIza"?
❌ Check 3: Is API enabled in Google Cloud Console?
❌ Check 4: Rate limit reached? (check quota at ai.google.dev)
```

### **Clerk Not Working**
```
❌ Check: Is NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY set?
❌ Check: Is CLERK_SECRET_KEY set?
❌ Check: Did you whitelist localhost:3000 in Clerk dashboard?
```

### **Strapi Admin Password Forgotten**
```bash
cd backend
npm run strapi admin:create-user
# Follow prompts to create new admin
```
